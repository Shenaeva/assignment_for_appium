# assignment_for_appium

Тестовое задание по мобильной автоматизации.

## Стек
- Java 22
- Maven
- Appium
- Selenide
- JUnit 5
- Android Studio
- Android Emulator (Android 11 / API 30)

## Что реализовано
Проект содержит автотесты для мобильных приложений:
- **VK Видео** (`com.vk.vkvideo`) — проверка сценария воспроизведения видео
- **Алхимия: Головоломка** (`com.ilyin.alchemy`) — сценарий получения подсказок за просмотр рекламы

В проекте используется Page Object подход. Выбор приложения сделан через тест: каждый тест сам указывает, какое приложение нужно запускать, без ручной смены кода перед запуском.

## Структура проекта
- `config` — конфигурация драйвера и выбор приложения
- `core` — базовый класс тестов
- `pages` — Page Object классы
- `tests` — тесты для VK Видео и Алхимии

## Требования
Перед запуском должны быть установлены и запущены:
- Android Studio
- Android Emulator с Android 11 / API 30
- Appium Server

## Запуск тестов

Запуск из IntelliJ IDEA:

открыть проект
запустить нужный тестовый класс из папки tests

Запуск через Maven:
```bash
mvn test
```
## Тестовое задание
### 1. VK Видео

Проверить, что выбранное видео воспроизводится, и обработать оба сценария:

позитивный — видео воспроизводится
негативный — видео не воспроизводится
### 2. Алхимия: Головоломка

Проверяем сценарий:

Нажать Играть  
Открыть подсказки  
Получить подсказку за просмотр рекламы  
Проверить, что количество подсказок увеличилось и итоговое значение равно 4  

#### Примечание
Сценарий с rewarded ad в приложении Алхимия может зависеть от конкретного рекламного флоу, который показывается приложением во время запуска теста.
